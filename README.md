# ECG Anomaly Detection with Autoencoders

Project developed for the **Machine Learning 2** course exam at  
**Università degli Studi di Napoli Parthenope**.

**Topic:** Autoencoders  
**Domain:** Biomedical signal processing  
**Task:** Unsupervised ECG heartbeat anomaly detection

---

## Overview

This project implements an anomaly detection pipeline for ECG heartbeats
using autoencoders trained only on normal samples.

At inference time, each heartbeat is reconstructed and its Mean Squared
Error (MSE) is used as an anomaly score:

- normal beats should have lower reconstruction error;
- anomalous beats should have higher reconstruction error.

The study focuses on four aspects:

- random versus patient-independent splitting;
- normalization strategy;
- Conv1D versus Dense autoencoders;
- Conv1D bottleneck size.

The main contribution is methodological: preprocessing and evaluation
choices can strongly affect the apparent performance of ECG anomaly
detection models.

---

## Dataset

The project uses the
[MIT-BIH Arrhythmia Database](https://physionet.org/content/mitdb/)
from PhysioNet.

Each heartbeat is extracted as a window of 360 samples centered on the
annotated R-peak:

```text
180 samples before the R-peak
180 samples after the R-peak
```

The annotations:

```text
N, L, R, e, j
```

are treated as normal. All remaining beat types are treated as anomalous.

| Category | Beats |
|---|---:|
| Normal | 90,320 |
| Anomalous | 15,589 |
| Total | 105,909 |

Dataset arrays are stored outside the repository and are regenerated by the
pipeline when required.

## Evaluation protocols

### Random split

Normal beats are randomly divided into training and test sets without
considering patient identity. All anomalous beats are included in the test
set.

| Subset | Samples |
|---|---:|
| Normal training | 72,256 |
| Normal test | 18,064 |
| Anomalous test | 15,589 |
| Total test | 33,653 |

This split may contain beats from the same record in both training and test
sets, so it is treated as a naive baseline.

### Inter-patient split

The main evaluation follows the standard DS1/DS2 patient-independent
protocol.

- training uses normal beats from DS1;
- testing uses normal and anomalous beats from DS2;
- no record is shared between training and test;
- paced records 102, 104, 107, and 217 are excluded.

| Subset | Samples |
|---|---:|
| Normal training | 45,841 |
| Normal test | 44,235 |
| Anomalous test | 6,908 |
| Total test | 51,143 |

## Normalization

The final pipeline uses global z-score normalization fitted only on normal
training beats.

```text
raw beats
    ↓
create split
    ↓
fit mean and standard deviation on normal training beats
    ↓
apply the same statistics to train and test
```

```text
x_normalized = (x - training_mean) / training_std
```

No test sample or test label is used to estimate normalization statistics.

| Split | Mean | Standard deviation |
|---|---:|---:|
| Random | -0.3486202365 | 0.4333013348 |
| Inter-patient | -0.4195560067 | 0.4502130157 |

Metadata is stored in:

```text
results/preprocessing_metadata.json
```

## Normalization ablation

Three normalization strategies were evaluated in isolated validation
experiments.

| Strategy | Random AUC | Inter-patient AUC | Observation |
|---|---:|---:|---|
| Per-beat | 0.4221 | 0.3824 | Ranking inversion |
| Legacy per-patient | 0.7948 | 0.8072 | Uses test-patient information |
| Global training-only | 0.6850 | 0.6458 | Valid ranking without test statistics |

Per-beat normalization removed information useful for anomaly ranking and
produced AUC values below 0.5.

The legacy per-patient implementation achieved higher AUC values, but used
normal beats from each test patient to estimate that patient's statistics.
This oracle assumption is not used in the final pipeline.

The global values above come from an isolated validation script. The final
results below come from a separate pipeline execution with the random seed
reset independently before every run.

The ablation outputs are available in:

```text
experiments/normalization_ablation.py
experiments/global_normalization_experiment.py
results/normalization_ablation.json
results/global_normalization_experiment.json
```

## Models

### Conv1D autoencoder

The convolutional encoder reduces the temporal dimension as follows:

```text
360 → 180 → 90 → 45
```

The decoder reconstructs the original 360-sample heartbeat using transposed
convolutions.

### Dense autoencoder

The Dense baseline uses fully connected encoder and decoder layers.

Both architectures are trained exclusively on normal beats using MSE
reconstruction loss.

## Main findings

### Random versus inter-patient split

Using the same 64-channel Conv1D architecture:

| Protocol | AUC |
|---|---:|
| Random | 0.7763 |
| Inter-patient | 0.6627 |

The observed AUC difference is:

```text
0.7763 - 0.6627 = 0.1136
```

The random split therefore produced a more optimistic estimate than the
patient-independent protocol.

This difference should not be interpreted as an exact causal measure of
patient leakage because the test sets also differ in patient and class
composition.

### Conv1D versus Dense

| Architecture | AUC | Precision | Recall | F1 |
|---|---:|---:|---:|---:|
| Conv1D | 0.6627 | 0.2247 | 0.5808 | 0.3241 |
| Dense | **0.8035** | **0.5241** | **0.5816** | **0.5514** |

The Dense autoencoder outperformed Conv1D under patient-independent
evaluation.

A plausible explanation is that the beats are already aligned around the
R-peak. This reduces the temporal variability that would normally favor a
convolutional architecture. This interpretation is plausible, but is not
established causally by the current experiment.

### Bottleneck sensitivity

| Bottleneck | AUC |
|---:|---:|
| 16 | **0.6850** |
| 32 | 0.6119 |
| 64 | 0.6627 |
| 128 | 0.6523 |

The relationship between bottleneck size and anomaly-detection performance
is non-monotonic. The narrowest tested bottleneck obtained the highest
Conv1D inter-patient AUC.

A lower reconstruction loss does not necessarily imply better anomaly
ranking.

## Evaluation methodology

For each run:

1. reconstruction MSE is computed for every test heartbeat;
2. reconstruction error is used as the anomaly score;
3. ROC-AUC is calculated;
4. Youden's J statistic selects an exploratory threshold;
5. precision, recall, F1, and confusion matrix are computed;
6. ROC and Precision-Recall curves are generated.

### Threshold limitation

The Youden threshold is selected using test labels.

Therefore:

- ROC-AUC is the primary threshold-independent metric;
- precision, recall, F1, and confusion matrices are exploratory;
- threshold-dependent metrics are not deployment-calibrated.

A production system should calibrate the threshold on a separate validation
set.


## Reproducibility

The project uses seed 42 for Python, NumPy, PyTorch, the random split, model
initialization, and DataLoader shuffling.

The seed is reset before each experimental run, while each training
DataLoader uses its own seeded generator. Results are therefore independent
of the run order in `configs/experiments.yaml`.

A targeted rerun of `conv_interpatient` reproduced the same epoch losses and
final metrics after deleting only its checkpoint.

Deterministic PyTorch settings are enabled where supported. Exact bitwise
reproducibility across different hardware and software versions is not
guaranteed.

---

## Limitations

- Evaluation is performed on a single ECG database.
- All non-normal beats are grouped into one anomaly class.
- No external dataset validation is included.
- Threshold-dependent metrics use a test-selected threshold.
- Results are based on one fixed seed rather than repeated seeded runs.
- No clinical claim is made.
