# ECG Anomaly Detection with a 1D Convolutional Autoencoder

Project for the Machine Learning course exam — Università Parthenope.

**Topic:** Autoencoders
**Scope:** Biomedical Signal Processing

## Overview

This project implements an unsupervised anomaly detection pipeline for ECG
heartbeats. A 1D convolutional autoencoder is trained **only on normal
heartbeats**. At inference time, the model is expected to reconstruct normal
beats accurately, while reconstructing anomalous beats (arrhythmias) poorly.
The reconstruction error is therefore used as an anomaly score.

## Why an autoencoder for this task

Labeling every possible arrhythmia type is expensive and requires expert
cardiologist annotation for every class. By training only on the majority
class (normal beats), the model learns a compact representation of what a
"normal" heartbeat looks like, without needing balanced or fully labeled data
for every anomaly type. This is a standard approach in biomedical anomaly
detection literature.

## Why a convolutional architecture (Conv1D) instead of a Dense one

An ECG beat is a temporal signal with a characteristic local morphology (the
QRS complex, P wave, T wave). 1D convolutions are designed to capture local,
repeating patterns along a sequence via weight sharing, which makes them more
robust to small temporal shifts (jitter) in the R-peak alignment and to
morphological variability between patients, compared to a fully-connected
(Dense) autoencoder that treats every sample as an independent feature.

## Dataset

**MIT-BIH Arrhythmia Database** (PhysioNet), 44 half-hour ECG recordings
sampled at 360 Hz, with beat-by-beat annotations made by cardiologists.

- Source: https://physionet.org/content/mitdb/
- Beats are extracted as fixed-length windows (360 samples, 180 before and
  180 after each annotated R-peak).
- Labels follow the AAMI standard: beats annotated as `N`, `L`, `R`, `e`, `j`
  are treated as **normal**; all other annotation symbols are treated as
  **anomalous**.
- Data is stored outside the project folder, in a shared external storage
  path (`/storage/internal_02/sperrotta-data/datasets`), and is not
  versioned in this repository (see `.gitignore`). It is downloaded and
  regenerated by running the data preparation script.

## How to reproduce

1. Install dependencies:
```bash
   pip install -r requirements.txt --break-system-packages
```

2. Download the dataset and extract heartbeats:
```bash
   python scripts/01_data_prep.py
```
   This downloads the 44 MIT-BIH records and saves `X_beats.npy` and
   `y_labels.npy` to `/storage/internal_02/sperrotta-data/datasets`.

3. Preprocess and split the data:
```bash
   python scripts/02_preprocessing.py
```

4. Train the autoencoder:
```bash
   python scripts/04_train.py
```

5. Evaluate on the test set:
```bash
   python scripts/05_evaluate.py
```

## Model architecture (summary)

A symmetric 1D convolutional encoder-decoder:

- **Encoder:** stacked `Conv1d` layers (with stride or pooling) progressively
  reducing the temporal dimension down to a compact latent representation.
- **Decoder:** stacked `ConvTranspose1d` layers reconstructing the original
  360-sample signal from the latent representation.
- **Loss:** Mean Squared Error (MSE) between input and reconstructed signal.
- **Training set:** normal beats only.
- **Test set:** normal and anomalous beats (for evaluation only).

(Full layer-by-layer configuration is documented in `scripts/03_model.py`.)

## Evaluation methodology

- The reconstruction error (MSE per beat) is computed on the held-out test
  set, which contains both normal and anomalous beats.
- A decision threshold on the reconstruction error is chosen (e.g. based on
  the error distribution on normal beats, or ROC-based optimization).
- Beats with reconstruction error above the threshold are flagged as
  anomalous.
- Metrics reported: ROC curve, AUC, precision, recall, confusion matrix.

Results and plots are saved in `results/`.

## Notes on dataset imbalance

The MIT-BIH dataset is naturally imbalanced (most beats are normal). This is
not a limitation for this approach, since the autoencoder is trained
exclusively on the normal class; the imbalance is only relevant at
evaluation time, where standard imbalance-aware metrics (ROC/AUC,
precision/recall) are used instead of raw accuracy.
